COMMON ISSUES & SOLUTIONS
━━━━━━━━━━━━━━━━━━━━━━━━━

"Health check returns 503"
→ Check PostgreSQL: docker ps | grep postgres
→ Check Redis: docker ps | grep redis
→ View logs: docker logs backend
→ Likely cause: Services haven't finished initializing
→ Solution: Wait 30-60 seconds or increase start_period

"wait-for-db timeout"
→ Check database startup: docker logs postgres
→ Verify credentials in .env
→ Check logs: docker logs backend | grep "pg_isready"
→ Solution: Increase MAX_RETRIES in wait-for-db.sh

"Memory limit exceeded"
→ Check usage: docker stats --no-stream
→ View memory trend: docker stats
→ Solution: Increase deploy.resources.limits.memory
→ Restart: docker-compose down && docker-compose up

"Logs not appearing"
→ Check log level: grep LOG_LEVEL .env
→ Verify JSON output: docker logs backend | python -m json.tool
→ Check log aggregation service
→ Solution: Verify LOG_LEVEL ≥ call level (DEBUG > INFO > WARNING)

"Graceful shutdown slow"
→ Check pending requests: Watch logs during shutdown
→ Check DB locks: docker exec postgres pg_locks
→ Solution: Increase stop_grace_period if needed

→ Full troubleshooting: See QUICK_REFERENCE.md
